# Contributing to SLR Agentic Tool

Thank you for your interest in contributing.

## Before you start

Open an issue describing what you want to change and wait 
for maintainer feedback before writing code.

## Workflow

1. Fork the repository
2. Create a branch: `git checkout -b feat/your-feature`
3. Make your changes
4. Push and open a Pull Request against `main`
5. Wait for review and approval

## Branch naming

feat/     → new features
fix/      → bug fixes
docs/     → documentation only
chore/    → maintenance tasks

## What we accept

- Bug fixes in the Free tier pipeline
- Documentation improvements
- New bibliographic source adapters
- Performance improvements

## What we do not accept

- Changes to enterprise features (private)
- Changes to pricing or plan logic
- Dependency upgrades without discussion

## Code style

- JavaScript ES Modules (import/export)
- Async/await over callbacks
- Descriptive variable names in English
- Comments in English
